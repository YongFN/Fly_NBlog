安装

通过虚拟机安装，相关软件及镜像请百度，本案例使用Centos7

Linux文件系统结构

所有的文件和目录都被组织成以一个根节点开始的倒置的树状结构

所有文件都存在于根目录下的不同目录中，不同的目录主要用于存放不同类型的文件

当前目录：./

上一层目录：../

默认不查找.开头的文件，例如.env文件

主要目录

目录 作用
/bin 存放系统常用的命令和可执行文件
/boot 存放系统启动配置文件和内核镜像
/dev 存放系统设备文件
/etc 存放系统配置文件
/home 用户的主目录
/lib 存放共享程序库文件
/media 存放可移动设备挂载点
/mnt 存放临时文件系统挂载点
/opt 第三方应用程序安装目录
/root root用户的根目录
/sbin 存放只有管理员才能运行的命令
/srv 存放特定服务的数据
/tmp 存放临时文件
/usr 存放系统范围的应用程序和数据
/var 存放可变数据，如日志文件

命令行操作

常用命令

命令 作用
cd 切换目录
ls 列出当前目录中的文件 -a 显示所有文件; -l 显示文件详细信息
touch 新建文件 -m 修改文件的修改日; -a为文件添加或修改访问时间; -t指定文件时间戳
mkdir 新建目录
rm 删除文件；-r 递归删除文件夹和所有子文件; -f 强制删除; -i 询问用户是否确认
cp 复制文件 -r 递归复制; -f 强制覆盖; -i 询问用户
mv 移动或重命名文件 -f、-i
cat 将文件内容输出到终端
echo 输出文本到终端
grep 在文件中查找匹配的文本
chmod 修改文件权限
pwd 查看当前完整路径

chmod：

  • u/g/o/a: 分别指定权限修改对象为[用户]/[组]/[其他人]/[所有人]
  • +/-: 增加或取消权限
  • r/w/x: 分别代表读取、写入、执行权限
  • 数字模式：chmod 644 test.txt，将 test.txt 文件权限修改为 -rw-r--r--

用户账号和权限管理

命令 作用
useradd 创建用户账号
passwd 修改用户密码
usermod 修改用户账号信息
userdel 删除用户账号
groupadd 创建用户组
groupmod 修改用户组信息
groupdel 删除用户组
chmod 修改文件或目录权限

软件包管理

命令 作用
apt-get Debian和Ubuntu中的软件包管理工具
yum Red Hat、Fedora和CentOS中的软件包管理工具
dnf 新一代Fedora中的软件包管理工具
pacman Arch Linux中的软件包管理工具
  1. yum install [package_name]: 安装指定的软件包
  2. yum update [package_name]: 更新已安装的软件包，找不到则更新全部
  3. yum remove [package_name]: 从系统中卸载已安装的软件包
  4. yum info [package_name]: 显示指定软件包的详细信息，包括版本，大小，发布日期等
  5. yum search [query]: 搜索可以安装的软件包。
  6. yum list [package_name]: 列出已安装的软件包
  7. yum upgrade [package_name]: 升级已安装软件包的最新版本
  8. yum clean all: 清理所有的yum缓存及rpm数据库中的旧数据
  9. yum groupinstall [group_name]: 安装指定的软件包组
  10. yum clean packages: 清除缓存目录下的软件包
  11. yum clean headers: 清除缓存目录下的 headers
  12. yum clean oldheaders: 清除缓存目录下旧的 headers
  13. yum clean, yum clean all (= yum clean packages; yum clean oldheaders) :清除缓存目录下的软件包及旧的 headers

软件服务管理

service

命令可以启动、停止或重启系统服务。

  • service [service_name] [command]

systemctl

systemctl是Linux中最新的服务管理工具。它可以将服务状态更精确地监控和控制，可以在系统引导时自动启动服务

例：systemctl start service_name

  • start
  • stop
  • restart
  • #启用一个服务，在系统引导时自动启动
    enable
  • #禁用一个服务，系统引导时不启动
    disable
  • status

chkconfig

chkconfig工具是一种管理服务的方式，它可以在系统启动时自动启动服务

打包和压缩

zip

zip：压缩文件或目录

选项-r：压缩目录

unzip：解压缩

选项-d：指定解压缩位置，默认在当前目录

gz

不能打包

选项：

  • -c ：将压缩数据输出到标准输出中，可以用于保留源文件
  • -d：解压缩
  • -r：压缩目录（除了目录本身）

gunzip：解压缩

bz2

bzip2：不能压缩目录

选项：

  • -d：解压缩
  • -k：压缩时保留源文件
  • -v：显示压缩的详细信息

bunzip2：解压缩

tar

打包不压缩

tar打包：

  • -c：打包
  • -f：指定压缩包的文件名
  • -v：显示打包过程

tar解打包：

  • -x：解打包
  • -f：指定压缩包文件名
  • -v：显示解包过程

tar.gz和tar.bz2

使用tar

选项：

  • -z：压缩和解压缩.tar.gz
  • -j：压缩和解压缩.tar.bz2
